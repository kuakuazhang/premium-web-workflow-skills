02 / Workflow

流程从“写 HTML”改成“设计生产”。

Codex 可以继续写代码，但不能跳过视觉方向。新的顺序把最容易失败的首屏放到最前面处理。

01

母版契约

先确定视觉 archetype，写清 Hero 资产、媒体语法、动效和 DOM/图片边界。

02

Hero 主视觉

默认用 image_gen 或真实资产做首屏视觉主体；不能用大字、网格和发光冒充。

03

真实结构

标题、按钮、表格、图表、筛选、流程图全部变成真实 DOM/SVG/canvas。

04

动效连接

用滚动、悬停、筛选、状态切换建立连续体验，而不是装饰性漂浮物。

05

截图门禁

桌面和移动端首屏都要过关；如果还像 PPT，就先重做 Hero。

03 / Implementation Boundary

图像负责氛围，网页负责信息。

这不是“禁止贴图”。真正的问题是把按钮、表格、图表、文字和业务逻辑都烤进图片里。新的规则要求：Hero、产品渲染、场景、材质可以是图；知识、交互和结构必须是网页。

04 / Anti-PPT Rules

不是更多装饰，而是更硬的失败判定。

另一个线程可以继续使用 Build Web Apps，但它不能绕过这些判断。只要命中下面的问题，就应该重做，而不是继续往下堆卡片。

Failure 01

巨大标题不是 Hero。

首屏不能靠超大文字和密集结论撑起来。标题要短，细节移到下一屏，画面需要真实视觉主体。

Failure 02

右图卡片会变 PPT。

Hero 图片应该成为全屏空间、背景 plate 或空间层，而不是被困在右侧圆角框里。

Failure 03

截图拼贴不算网页。

参考图要被消化：按钮变按钮，表格变表格，图表变 SVG/canvas，图像只保留视觉资产。

05 / Quality Gate

交付前先打分。低于 4 分，重做首屏。

网页审美不能只靠“感觉”。新的 Skill 会从 Hero、视觉主体、文字密度、母版复现、真实结构、交互和移动端首屏七项检查。

4+Hero immersion
4+Visual subject
4+Text density
4+Archetype fidelity
4+DOM/SVG rebuild
4+Useful motion
4+Mobile viewport
