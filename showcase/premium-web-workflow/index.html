Reference First.Build Better.

A Codex web workflow that starts from a chosen premium reference, then turns the mechanics into hero assets, real HTML structure, motion, and screenshot gates.

Install Skill Choose Reference

01 / Reference Control

Choose.Then Build.

The workflow should not silently invent a premium style. It must first declare whether the page is using a user-selected MotionSites prompt, a Godly real-site reference, or a no-reference experiment.

Direct reuse mode

MotionSites prompt first

When the user pastes an official copied prompt, Codex reproduces the mechanics first: video role, glass buttons, nav behavior, entry timing, content position, and responsive rules.

Deep reference mode

Godly mechanics

Inspect a real site, extract layout, scroll, motion, typography, and asset grammar, then rebuild those mechanics for the user's content.

Risk label

No silent default

If no reference is selected, say so before coding. Do not imply a Godly or MotionSites result after the fact.

02 / Hero Production

Hero is nota title block.

The first viewport is a separate deliverable. It needs a visual subject, restrained copy, a clear text-safe area, and desktop plus mobile screenshot review before the rest of the page is accepted.

Image, video, or scene first. For public pages, default to GPT image generation or owned media. SVG diagrams are useful later, but not as a cheap replacement for the primary Hero visual.
Generated imageUse for cinematic plates, product renders, editorial scenes, textures, and fallback posters.
Owned videoUse when motion is the point. Localize critical video files instead of depending on temporary URLs.
Real HTML layerTitles, buttons, tabs, labels, charts, and explanations stay in the page, not inside the raster asset.

03 / Build Pipeline

Keep the vibe.Fill the page.

A copied Hero prompt is only the opening scene. The rest of the page must continue the same system with real content, not stop because the reference only provided one viewport.

04 / Quality Gate

Reject thePPT feeling.

The workflow needs hard failure checks. A page fails when it only changes colors, keeps the same composition, hides content below an empty Hero, or turns every section into the same glass card.

05 / Install

Make this thedefault habit.

When the user says "做网页", Codex should ask for or declare the reference path, build the Hero as a real design object, continue the same visual system through the content, and verify before shipping.
