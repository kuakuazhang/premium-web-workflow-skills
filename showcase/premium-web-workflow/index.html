先选参考再做网页

让 Codex 先从高级参考出发，再把其中的构图、主视觉、真实结构、动效和截图验收转成可交付网页。

安装 Skill 选择参考

01 / 参考控制

先选参考再动代码

工作流不能静默发明所谓高级感。它必须先说明：这次是用户选择的 MotionSites prompt、Godly 真实网站参考，还是无参考实验。

直接复刻模式

先还原 MotionSites

用户粘贴官方复制 prompt 时，Codex 先复刻机制：视频角色、玻璃按钮、导航行为、入场节奏、内容位置和响应式规则。

深度参考模式

拆解 Godly 机制

检查真实网站，把布局、滚动、动效、字体和图片语法拆出来，再用我们的内容重建。

风险标签

不能默认开干

如果没有选择具体参考，编码前就要说清楚。不能事后暗示自己做的是 Godly 或 MotionSites 风格。

02 / Hero 生产

Hero 不是标题块

第一屏是独立交付物。它需要视觉主体、克制文案、明确的文字安全区，并且要先通过桌面和移动端截图检查。

先有图像、视频或场景。 公开页面默认用 GPT 生图或已有素材做主视觉。SVG 可以做图表和结构，但不能偷换成廉价 Hero 插画。
生成图片用于电影感底图、产品渲染、编辑场景、材质纹理和视频备用封面。
自有视频当动效本身有价值时使用。关键视频要本地化，不能长期依赖临时远程地址。
真实网页层标题、按钮、标签、图表和说明必须留在 HTML 中，不能烤进图片里。

03 / 构建流程

保留气质填满内容

复制来的 Hero prompt 只是开场。后面的页面必须沿用同一套视觉系统，把我们的真实内容继续做完，而不是停在一个漂亮首页。

01

选参考

用户选择、Agent 声明，或者明确标注这是无参考实验。

02

做 Hero

第一屏按照选定参考的机制来做，不再自由发挥。

03

填内容

把同一套风格延展到所有 section，而不是换成报告模板。

04

加动效

用入场、悬停、菜单和滚动节奏连接体验。

05

截图验收

桌面和移动端都要看。如果回到旧模子，就重做。

04 / 质量门禁

拒绝PPT 感

这个工作流必须有硬失败标准。只换颜色、保留同一套构图、首页下面没有内容、或者所有 section 都变成同一种玻璃卡片，都算失败。

4+

参考还原

选定参考的机制必须看得见，而不是只在嘴上说。

4+

Hero 沉浸

第一屏有真实视觉主体，也有清晰入口。

4+

内容完整

用户原本要表达的内容必须真的出现在页面里。

4+

移动端首屏

不能有标题裁切、信息挤压或按钮失效。

05 / 安装使用

把审美变成默认

当用户说“做网页”，Codex 应该先询问或声明参考路径，把 Hero 当成真实设计对象，再把同一套视觉系统延展到内容区，最后截图验收。
