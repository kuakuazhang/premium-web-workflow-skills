Environment Report

Awwwards / Environment Report

反 PPT
生产报告

这轮训练行业报告和可视化网页：首屏必须有真实数据对象，正文必须用矩阵、关系图和时间线承载判断，而不是报告封面加卡片区。

桌面优先真实 SVG / DOM蓝绿黄研究气质
核心判断：网页失败不是 CSS 问题。它通常是参考过浅、首屏没有视觉主体、内容区骨架偷懒、验收没有截图证据。

01 / Signal Map

把失败
画出来

参考 Environment Report 的 single-page 数据叙事，本轮用一个真实 SVG 信号图替代“报告页标题 + 几张卡片”的惯性。

蓝线：设计生产力从具体参考开始，页面越早形成骨架，后面越少靠补丁救火。
绿线：内容完整度只做首屏会快速下滑；内容区必须沿用同一视觉系统。
黄线：证据质量截图、自评分、失败点是写回 Skill 的前置条件。

02 / Matrix

失败矩阵
不靠感觉

报告型网页最容易“看起来有信息，但没有结构”。矩阵用真实表格呈现，不把判断烤进图片。

问题表象真正原因修法
像 PPT封面大标题 + 数据卡片没有参考网页骨架，只是在排版文字先找 Awwwards 具体原站，拆首屏和滚动结构
Hero 空洞网格、粒子、超大字没有视觉主体或真实场景生成图像、产品对象、数据对象或代码场景
可写回截图分数稳定超过 4 分规则经过了实验验证阶段结束后合并到 Skill，而不是每轮立刻写死

03 / Timeline

证据后
再写回

Skill 的进化不能靠聊天里的情绪判断。每条规则都要有候选、参考、截图、失败点和复盘依据。

Step 01

候选

每轮 3 个 Awwwards 具体网页。

Step 02

主参考

选 1 个原站，写清选择理由。

Step 03

实验页

真实内容进入新骨架。

Step 04

截图

桌面首屏 + 两个内容区。

Step 05

写回

只保留被证据支持的规则。
